Global File Settings

This screen allows you to set the names of the files that are created by the Data Archiver. There are two sections.

The top portion of this screen is for the Network Shot Parameter File. The Data Archiver can combine the parameters from each shot on all the machines into a single file. The purpose of this function is to make it easy for customers to analyze the shot parameter data using a program like Excel.

File Name:

This is the path to the file that will be created for the shot parameter records. If you empty this or set it to UNKNOWN then no file will be created.

Record Format:

 This is the format of the record that is created when the program receives a shot from one of the machines being monitored. You can specify what goes into this record and where it goes.

The bottom portion of the screen allows you to name the zip files created by the Data Archiver Backup program. There are two types of files.

Normally you would use one setting, one pair of names, for all the machines and specify enough information in the file names to differentiate the files. To do this, make sure the Use a different setting for each machine checkbox is NOT checked.

If that checkbox is checked then the listbox of machine names is made active and you can specify an entirely different filename for each machine.

Note that when you uncheck the Use a different setting for each machine checkbox the default filename is presented to be edited. This filename will be used for any new machines that are added later until you change the names for that machine.

Part Results File:

This is the file that is created when the program reaches one of the criteria (number of shots received or time of day) for zipping the shots. All of the saved shots and parameters are saved in this file.

Machine Results File:

 This is the file that is created at the same time as the Part Results file but only if there has been a change in the downtime status since the data was saved last.

Mirror Drive Settings

If you want to automatically copy the backup zip files to a second drive you can specify a mirror drive. In order to do this you must edit the c:\v5ds\exes\dsbackup.ini file and insert the following three lines at the beginning of the file. 

[Config]
HaveMirror=Y
MirrorRoot=E:
The MirrorRoot is the root directory of the second drive. This does not have to be the actual root. If you are currently saving the zip files in the c:\v5backup directory and the MirrorRoot is 
MirrorRoot=E:
then the mirror files will be saved in the E:\v5backup directory.

If the MirrorRoot is 

MirrorRoot=E:\vtw
then the mirror files will be saved in the E:\vtw\v5backup directory.

You can also specify a network directory as the save location 

MirrorRoot=\\vtw-backup-computer
This will save the mirror files in the v5backup directory of the computer named vtw-backup-computer.

Note that in order to save to a network directory that directory, v5backup in this case, must already exist and be shared with full access.
