Version 8 Data Base

  • Database files end in .txt and are fixed length with the fields separated by semicolons (;). Strings are enclosed with "double quotes".
  • Both CSV and Setpoint files end in .csv. CSV files are multiple record files where each record has a number of fields, each separated with a comma. They are just like database files except they are not fixed length, they have comma's instead of semicolons, and don't have quotes around the strings, since everything is just text anyway.
    Setpoint files are made up of multiple records but each record (line) is a setpoint for a different variable. The format is description + comma + value for each record. For example, the plookup file has the shot number and the time of the last shot for a part. A file would look as follows,
    Shot Number,15
    Date,09/24/2006
    Time,12:45:34
    Update Status,0
  • There are also some files that end in .dat or even .txt that I would rather not admit to but must. They are just free form files with one variable per line. An example is the ftii_global_settings.txt file which is just a list of variables for the ft2 board. Each line is like
    V319=213
    which is the format of the set variable commands in the ft2 control file.
  • Before I settled on the Setpoint files I used .ini files for at least one set of data. An ini file is an old windows thing. I use them instead of the registry to save startup settings but I also have the FTII_EDITOR_SETTINGS.INI file which has machine-based settings. Each line is a name followed by an = followed by the value.

There are a couple of rules. I keep a file called semaphor.txt in the machine directory and get a full lock on this before making changes to the machine or part setups or before saving new shot data. If I can't open a file because someone else has locked it I try again at an ever decreasing pace until I finally give up. The only files that I keep open for more than a fraction of a second are the shotparm.txt (shot parameters) and graphlst.txt (list of shots) files that are found in the part results directory. I'm planning on doing away with the graphlst.txt file as soon as I write a routine to load a dynamic list box with directory list results. I also keep the downtime.txt file open with a write lock, so you can view it read-only, while you are editing the downtime values. This is located in the machine results directory.

PLOOKUP.CSV (setpoint file)


This file shows the last shot number saved and the time. It also (as of version 7.14) contains the update status. This contains bits that indicate whether a certain type of update has occured. The only bit currently used is 1 which indicates that the Adam Thermocouple values have been saved. This is used to prevent multiple saves if I get more than one reading per shot.
The Last Shot Name is the last one used. The shot number in the last shot name may not agree with the Last Shot Number as this can be reset change the next shot name. The "Update Status" is for the
Last Shot Name string
Last Shot Number integer
Date mm/dd/yyyy
Time hh:mm:ss
Update Status 0

SHOT_NAME.CSV (setpoint file)

This file contains the settings that allow me to create a shot name.
The first entry is always the length of the shot name field. This is the max number of characters that will be needed. It does NOT include the beginning and ending quotes.
The second entry is always the pattern; This tells how to make the shot name. At the present you must have the shot number, {s}, or {s4}, etc, somewhere in the name.
The third entry is the path pattern. This defines how to calculate the subdirectory for the shot results. This entry is relative to the part results directory - e.g. {yyyy}-{mm} would create a new directory every month like c:\v5\data\e20\visitrak\results\2011-01\. This is not implimented yet.
All entries after this are not in any particular order and allow the user to create standard fields that can be set.
For Example: Model,T36 would define a tag {Model} with a value T36. If the pattern contains {Model} it will be replaced with "T36" when the shot name is created.
Field Length 16
Pattern {s6}
Path{yyyy}\{mm}\
User Defined User Defined
