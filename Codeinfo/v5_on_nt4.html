Windows NT Problems

The main problem with Windows NT is that you must set up a user account for each computer that is going to access a shared directory.

For example, a monitoring computer communicates with both WS0 and FS. If the monitoring computer is an NT system then you must set up a user account for each of these.

The user account is set up using the Login Name and Password of the other computers. Let me use the Data Server as an example.

Normally we set the Primary Network Logon of the Data Server (in the network properties) to Windows Logon so that the computer does not ask for a logon when it boots up. You can find out what the current logon is by choosing to log in as another user. On most Microsoft systems (except for 98) you do this by choosing Shutdown from the Start menu and then choose Close all programs and log on as a different user.

In order for windows to log you in automatically you must have the password set to nothing.

If you log in as FS with no password then in order for you to access the files on the NT monitoring computer you must create a new user on the monitoring computer (Start, Programs, Administrative Tools (Common), User Manager). The new user must have a username of FS and no password. I usually make the user a power user and make sure to set the Password Never Expires checkbox.
