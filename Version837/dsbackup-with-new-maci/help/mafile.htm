Machine Results File:

This works the same way as the Part Results File. The tags are replaced with the dynamic content and everything else is just copied. The tags are as follows.

TagContent
{c} Computer
{m} Machine
{dd} Day of the month [1,31]
{mm} Month [1,12]
{yyyy} Year [2002]
{yy} Year [02]
{hh} Hour [0,23]
{mi} Minutes [0-59]
{ss} Seconds [0-59]
{s} Shift Number

These are machine-based results, so it isn't appropriate to include the part name in the results.

As an example, for machine M01 the program program originally saved files with the following name (on 3/20/03 at 12:42 PM ). 

C:\V5BACKUP\M01_03_20_12_42.MAZ
The .MAZ extension is used for all Machine Results Files and is not a part of the name that you specify. To make the new name the same as the old one you would use the following. 
C:\V5BACKUP\{m}_{mm}_{dd}_{hh}_{mi}
Perhaps you would like to put the results for each day in a separate folder. You might use the following. 
C:\V5BACKUP\{mm}_{dd}\{m}_{hh}_{mi}
Which would create a new folder if necessary and would save the above file as 
C:\V5BACKUP\03_20\M01_12_42.MAZ
