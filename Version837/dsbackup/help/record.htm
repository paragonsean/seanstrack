Record Format

This works the same way as the filename. The tags are replaced with the dynamic content and everything else is just copied. The tags for the RecordFormat are as follows.

TagContent
{c} Computer
{m} Machine
{p} Part
{g} Good Shot Indicator: "A"larm,"W"arning,"G"ood
{n} Shot Number
{dd} Day of the month [1,31]
{mm} Month [1,12]
{yyyy} Year [2002]
{yy} Year [02]
{hh} Hour [0,23]
{mi} Minutes [0-59]
{ss} Seconds [0-59]
{s} Shift Number
{1} Parameter Value [1,60]

Thus the following record format 

{m},{p},{mm}/{dd}/{yyyy},{hh}:{mi}:{ss},{n},{1},{2},{3}
generates the followng entry for machine M02 and part VISITRAK. 
M02,VISITRAK,05/24/2002,15:06:56,4106,28.92,634.88,573.98
                                 ---- ----- ------ ------
                                  |     |      |      +-------- Parameter 3 Value (Slow Shot Time)
                                  |     |      +--------------- Parameter 2 Value (Total Shot Time)
                                  |     +---------------------- Parameter 1 Value (Fill Time)
                                  +---------------------------- Shot Number

The Good Shot Indicator displays whether the shot has an alarm, warning, or was a good shot. An "A" means there was at least one alarm. A "W" means there were no alarms but at least one warning. A "G" means there were no alarms or warnings. The word displayed for each of these levels can be changed. Unfortunately you must edit the dsbackup.ini file in order to do this. All of the settings are stored in c:\v5ds\exes\dsbackup.ini. There are three lines that can be added in the "Shotparm" section. 

HaveAlarmShot=Alarm
HaveWarningShot=Warning
HaveGoodShot=Good
These are just examples. You can put any words you wish at the end. For example, if you consider a shot with only warnings to be good you could change the warning word to: 
HaveWarningShot=Good
